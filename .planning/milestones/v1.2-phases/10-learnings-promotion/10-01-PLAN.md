9. Verification must cover ALL symptoms (ard 014). When fixing a bug reported with multiple symptoms, verify every single symptom is resolved, not just the headline.
10. API ConnectionRefused recovery via heartbeat polling (opengraphdb). Don't rely solely on events for detecting stalled sessions. Heartbeat polling catches ConnectionRefused and other non-event failures.

Learnings to promote:

1. **Codex launch syntax** (ryan 015). The correct flag is `--dangerously-bypass-approvals-and-sandbox`, NOT `--dangerously-skip-permissions`. Add to Troubleshooting or a new Codex subsection.

2. **Codex launch: use add+start+sleep+send, not launch** (ard 002, opengraphdb). `launch -c codex` frequently times out. Pattern: `agent-deck add -t "name" -c codex /path`, then `agent-deck session start "name"`, wait 3-5s, then `agent-deck session send "name" "message"`.

3. **Never do releases from conductor** (agent-deck 006). Conductor should never do git push, tag, or release directly. Launch a dedicated session for release work.

4. **Gemini sessions for video analysis** (ard 017). Use `-c gemini` for video review workflows. Gemini has native video understanding capabilities.

5. **--wait flag behavior** (ryan 006). `session send --wait` is available but sometimes exits with code 2. For critical flows, fall back to polling session output.

6. **-cmd flag breaks -group parsing** (ard 001). Known issue: using `-cmd` alongside `-group` flag can cause parsing problems. Always use `-c claude` or `-c codex` short form instead of `--cmd`.

7. **Use agent-deck rm to delete sessions** (ard 009). Command is `agent-deck rm "session-name"`. `session stop` sets status to error, it does not remove.

8. **Session identification: exact title > full ID > 8-char prefix** (opengraphdb). No fuzzy matching, no partial titles. Only these three resolution methods work.

9. **Worktrees only contain git-tracked files** (opengraphdb). Must copy untracked source files manually into worktrees if they are needed.

10. **launch -m may not reliably deliver initial message** (ryan 013). After launch with -m flag, verify the session transitioned from its initial state or confirm output changed before assuming delivery.

11. **Heartbeat messages too verbose for multi-conductor** (agent-deck 003). When running multiple conductors, heartbeat bridge messages create noise. Future improvement area.
