<objective>
Fix MCP socket proxy request ID collisions when multiple sessions share a proxy instance (#324).

Purpose: The SocketProxy multiplexes multiple client connections onto one stdio MCP process. Currently, client-supplied JSON-RPC IDs are used as routing keys, so two sessions sending `id: 1` collide, causing one session's response to be routed to the wrong caller or dropped entirely. This causes tool call hangs in production multi-session workflows.

Output: A fixed `socket_proxy.go` with atomic ID rewriting and session-scoped response routing, plus three integration tests verifying concurrent correctness under the race detector.
</objective>
