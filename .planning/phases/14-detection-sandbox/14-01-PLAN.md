<objective>
Remove all embedded `tmux set-environment` calls from command builder shell strings and rely exclusively on host-side Go `SetEnvironment()` calls after tmux session start.

Purpose: `tmux set-environment` inside Docker sandbox containers cannot reach the host tmux socket, silently failing and leaving session IDs unset (#266). The fix applies universally (not just sandbox) because host-side `SetEnvironment()` is idempotent for non-sandbox sessions and eliminates divergent code paths. The existing post-start `SetEnvironment` pattern at instance.go:1806 (AGENTDECK_INSTANCE_ID) and `SyncSessionIDsToTmux()` already demonstrate the correct approach.

Output: Cleaned command builders in instance.go with no `tmux set-environment` in shell strings. Pre-generated UUIDs for new Claude sessions. Host-side SetEnvironment calls after Start()/Restart(). Tests verifying the cleanup.
</objective>
