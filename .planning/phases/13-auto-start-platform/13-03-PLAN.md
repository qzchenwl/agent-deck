<objective>
Replace shell-level `uuidgen` dependency with a Go-side `generateUUID()` function using `crypto/rand`, and update `buildClaudeCommand` and `BuildForkCommand` to embed the pre-generated UUID directly in the command string. This eliminates the external `uuidgen` binary dependency (Mode E from research) that fails silently on minimal WSL/Linux installations where `uuid-runtime` is not installed.

Purpose: Resolves the Mode E failure mode of PLAT-01 (#311) where `uuidgen` not being installed causes `$session_id` to be empty, leading to `tmux set-environment CLAUDE_SESSION_ID ""` and `WaitForClaudeSession` timing out. As a side benefit, removing the `$(` subshell expression from the command string eliminates the trigger for the double-`bash -c` wrap in `tmux.go:Start()` at line 1177, simplifying the command chain.

Output: Modified `internal/session/instance.go` (new `generateUUID()`, updated command builders), updated tests in `internal/session/instance_test.go` and `internal/session/fork_integration_test.go`.
</objective>

    ```go
    // generateUUID returns a lowercase UUID v4 using crypto/rand.
    // Replaces shell-level: session_id=$(uuidgen | tr '[:upper:]' '[:lower:]')
    // This eliminates the external uuidgen binary dependency that may not be
    // installed on minimal WSL/Linux systems (requires uuid-runtime package).
    func generateUUID() string {
        b := make([]byte, 16)
        if _, err := rand.Read(b); err != nil {
            // Fallback to timestamp-based pseudo-UUID (extremely rare)
            return fmt.Sprintf("00000000-0000-0000-%016x", time.Now().UnixNano())
        }
        b[6] = (b[6] & 0x0f) | 0x40 // version 4
        b[8] = (b[8] & 0x3f) | 0x80 // variant bits
        return fmt.Sprintf("%08x-%04x-%04x-%04x-%012x",
            b[0:4], b[4:6], b[6:8], b[8:10], b[10:16])
    }
    ```
